## Overview

**Aegis-Beacon** is an open-source, ultra-low-cost emergency rescue beacon designed for avalanche survival, backcountry emergencies, and SAR (Search and Rescue) operations. It fits in a jacket pocket, costs around $12 to build, and can operate for **72+ hours** on a single 18650 cell.

v4.0 adds a 0.96" OLED screen for at-a-glance status without needing a serial monitor, and a 3.5mm audio jack that outputs pitch-variable alert tones in SEARCH mode вҖ” the closer you get to a signal, the higher and faster the tone.

The device operates in four modes, switchable in real time with a physical button:

| Mode              | LED        | OLED                        | Audio               | What it does                                                |
|-------------------|------------|-----------------------------|---------------------|-------------------------------------------------------------|
| рҹ”ҙ **BEACON**     | Red blink  | Freq + TX progress          | Morse click stream  | Transmits Morse SOS on 1вҖ“10 frequencies via OOK carrier     |
| рҹ”ө **SEARCH**     | Blue blink | Freq + RSSI bar + last hit  | Variable pitch beep | Scans all frequencies, logs RSSI, audio alert on detection  |
| вҡҷпёҸ **CONFIG**     | Both blink | SSID + IP + QR hint         | Silent              | WiFi AP + captive portal dashboard                          |
| вҡЎ **EMERGENCY**   | Red fast   | Full-screen inverted "вҡЎSOSвҡЎ" | Continuous tone    | Max power, continuous TX, no sleep                          |

All settings are stored in NVS (non-volatile storage) and survive reboots and deep sleep cycles. A **fail-safe** system ensures the device defaults to `433.500 MHz / "SOS" / 13 WPM / 17 dBm` if NVS is ever empty or corrupt.

## Features

### RF & Transmission

- вң… **OOK/CW mode** вҖ” carrier on/off keying, audible on any AM-mode scanner or SDR receiver
- вң… **Frequency hopping** вҖ” up to 10 configurable frequencies per cycle
- вң… **PARIS-standard Morse timing** вҖ” `dot=1u, dash=3u, inter-char=3u, word=7u`
- вң… **Configurable WPM** вҖ” 5 to 30 WPM via dashboard slider
- вң… **Configurable TX power** вҖ” +2 to +17 dBm (RA-02 hardware limit)
- вң… **Configurable repeat count** вҖ” 1вҖ“5 message repetitions per cycle
- вң… **Mid-TX interrupt** вҖ” mode button can abort a transmission instantly

### Search / Scan Engine

- вң… **Per-frequency RSSI measurement** with configurable dwell time (100вҖ“5000 ms)
- вң… **Configurable detection threshold** (-120 to -40 dBm)
- вң… **Rolling hit log** вҖ” last 20 detections stored in RTC RAM (survives deep sleep)
- вң… **Signal classification** вҖ” STRONG (вүҘ -60 dBm) / MEDIUM (вүҘ -80) / WEAK (вүҘ threshold)
- вң… **ASCII signal bar** printed to Serial on every scan pass
- вң… **OLED RSSI bar** updated every scan step
- вң… **LED alert** вҖ” blue LED blinks on detection, fast blink on strong signal

### Audio Alert System *(new in v4.0)*

- вң… **PWM audio output** on GPIO 18 via LEDC at 40 kHz carrier (above hearing), amplitude modulated to produce audible tones
- вң… **Signal-strength proportional tone:** silence вҶ’ slow beep вҶ’ fast beep вҶ’ continuous tone
- вң… **Pitch rises with signal strength:** 440 Hz (weak) вҶ’ 880 Hz (medium) вҶ’ 1760 Hz (strong)
- вң… **Morse click stream** in BEACON mode for earphone debug monitoring
- вң… **Configurable volume** (0вҖ“255 PWM duty, ~0вҖ“3.3 V peak)
- вң… **Master enable/disable** saved to NVS
- вң… **AC-coupled output** вҖ” 100О© series + 10 ВөF blocking cap, safe for 16вҖ“600 О© headphones

### OLED Display *(new in v4.0)*

- вң… **SSD1306 0.96" 128Г—64** via I2C (GPIO 0 SCL, GPIO 1 SDA)
- вң… **Mode-specific screen layouts** вҖ” distinct UI for BEACON, SEARCH, EMERGENCY, CONFIG
- вң… **Boot splash** вҖ” logo + version, 2 s
- вң… **BEACON screen:** frequency, TX progress, message, WPM, cycle number, sleep countdown
- вң… **SEARCH screen:** current frequency, RSSI bar chart, last hit, total hits, elapsed time
- вң… **EMERGENCY screen:** full-screen inverted display with large blinking "вҡЎSOSвҡЎ"
- вң… **CONFIG screen:** AP SSID, IP address, and "SCAN TO CONFIG" QR code hint
- вң… **250 ms refresh rate** вҖ” fast enough for live RSSI without CPU overhead
- вң… **Configurable enable/disable + invert** вҖ” save power or adapt to bright sunlight

### Dashboard & Configuration

- вң… **WiFi captive portal** вҖ” connect any phone/laptop, browser opens automatically
- вң… **Dark military-aesthetic UI** вҖ” Orbitron / Share Tech Mono fonts, responsive
- вң… **Live Morse preview** вҖ” message decoded to dots/dashes as you type
- вң… **Frequency manager** вҖ” add/remove up to 10 frequencies with band labels
- вң… **Audio controls** вҖ” volume slider + enable toggle
- вң… **OLED controls** вҖ” enable toggle + invert toggle
- вң… **Scan history panel** вҖ” RSSI bar charts, auto-refreshes every 5 s
- вң… **Device status panel** вҖ” boot cycles, TX/scan counts, free heap, uptime
- вң… **Test TX button** вҖ” sends single SOS burst from browser
- вң… **Test Scan button** вҖ” scans all frequencies live, returns RSSI per freq
- вң… **Emergency override** вҖ” activate max-power SOS from browser
- вң… **5-minute AP timeout** вҖ” auto-reverts to beacon mode if nobody connects

### Reliability & Safety

- вң… **Hardware watchdog** вҖ” 30 s WDT resets device if firmware hangs
- вң… **NVS fail-safe** вҖ” hardcoded defaults if storage is empty or corrupt
- вң… **Factory reset** вҖ” hold both buttons at boot for 5 s
- вң… **RTC RAM state** вҖ” mode, cycle counters, scan hits survive deep sleep
- вң… **Interrupt-driven buttons** вҖ” responsive even during TX delays
- вң… **Debounced button logic** вҖ” 50 ms hardware debounce + hold duration measurement
- вң… **Deep sleep** вҖ” ESP32-C3 draws ~10 ВөA between TX cycles

### Debug System

- вң… **Color-coded ANSI serial log** вҖ” `[INFO]`, `[OK]`, `[WARN]`, `[ERROR]`, `[SCAN]`, `[BTN]`, `[MODE]`, `[CFG]`, `[OLED]`, `[AUDIO]`
- вң… **Verbose mode** вҖ” `DEBUG_VERBOSE 1` adds per-symbol Morse timing + RadioLib state codes
- вң… **Per-scan ASCII bar graph** вҖ” visual RSSI display at 115200 baud
- вң… **Boot diagnostics** вҖ” reset reason, free heap, CPU freq, SDK version on every boot

### Schematic Notes

- The ESP32-C3 SuperMini has an **AMS1117-3.3 LDO** powered from the 5 V / VBUS rail. No external LDO required.
- The TP4056 module with DW01A includes over-charge, over-discharge, and short-circuit protection. No external BMS needed.
- RA-02 SPI lines must be **3.3 V only**. ESP32-C3 GPIOs are 3.3 V natively вҖ” no level shifting required.
- The ESP32-C3 does **not** have a hardware DAC. Audio is generated via LEDC PWM at 40 kHz (above hearing range), then low-pass filtered by the headphone impedance. The 10 ВөF capacitor blocks DC bias from the headphone drivers.
- OLED I2C address is **0x3C** by default. Some modules use 0x3D вҖ” change `OLED_ADDR` in the firmware if the display does not initialise.
- Solder the **17.3 cm wire antenna** vertically upward from the RA-02 ANT pad for best omni-directional radiation pattern.

## Operating Modes

### BEACON Mode

The primary emergency mode. On each cycle the device:

1. Disables WiFi and Bluetooth stacks (saves ~120 mA)
2. Iterates through all configured frequencies
3. On each frequency: initialises SX1276 in OOK mode, transmits the full Morse message (repeated N times)
4. Emits Morse click tones through the audio jack in sync with TX (if audio enabled)
5. Updates the OLED with current frequency and TX progress
6. Puts the SX1276 into sleep mode
7. Enters ESP32 deep sleep for the configured interval
8. Wakes up and repeats

**Morse timing (PARIS standard):**

| Element              | Duration        |
|----------------------|-----------------|
| Dot                  | `1200 / WPM` ms |
| Dash                 | `3 Г— dot` ms    |
| Intra-character gap  | `1 Г— dot` ms    |
| Inter-character gap  | `3 Г— dot` ms    |
| Word gap             | `7 Г— dot` ms    |

At 13 WPM: dot = 92 ms, dash = 277 ms. "SOS" takes ~2.7 seconds.

---

### SEARCH Mode

Continuous scan mode for locating other beacons. The device:

1. Disables WiFi and Bluetooth stacks
2. Loops through all configured frequencies indefinitely
3. On each frequency: opens FSK receive window, measures peak RSSI over the dwell period
4. Emits audio tones proportional to signal strength (see [Audio System](#audio-system))
5. Updates the OLED with current frequency, RSSI bar, last hit, and total detections
6. Logs all detections above the configured threshold to RTC RAM
7. Blinks the blue LED on detection (fast blink = strong signal)
8. Prints an ASCII signal bar graph to Serial for each frequency

**Sample serial output (SEARCH mode):**

```
[  42381][SCAN ] Scan pass вҖ” 3 frequencies
[  42382][SCAN ] [0] 433.500 MHz  RSSI=-112 dBm  |....................| quiet
[  42784][SCAN ] [1] 434.500 MHz  RSSI= -87 dBm  |#########...........| *** SIGNAL ***
[  42785][SCAN ] HIT recorded: 434.500 MHz -87 dBm [MEDIUM]  total hits=1
[  43188][SCAN ] [2] 435.000 MHz  RSSI=-109 dBm  |....................| quiet
[  43189][SCAN ] Pass complete вҖ” elapsed 0 s | total detections: 1
```

### CONFIG Mode (Dashboard)

Activated by holding SW_CONFIG (GPIO 21) for 3 seconds. The device:

1. Starts a WiFi Access Point (`AegisBeacon`, open/no password)
2. Starts a DNS server that redirects all domains to the device IP
3. Serves the dashboard at `http://192.168.4.1`
4. Displays SSID, IP, and "SCAN TO CONFIG" on the OLED
5. On most phones, a "Login to network" notification appears automatically (captive portal)
6. After saving, the device reboots into the selected mode
7. If nobody connects within 5 minutes, the device automatically reboots

**Connect from any device:**

- WiFi SSID: `AegisBeacon`
- No password required
- URL: `http://192.168.4.1` (or wait for the captive portal popup)

---

### EMERGENCY Mode

Activated by holding SW_MODE for 2 seconds, or via the dashboard's Emergency button. In this mode:

- Maximum TX power (+17 dBm)
- Message repeated 3Г— per frequency
- **No deep sleep** вҖ” continuous transmission with no pause
- OLED shows full-screen inverted "вҡЎSOSвҡЎ" animation
- Audio emits continuous tone
- Persists across power cycles (stored in RTC RAM)
- Reset by entering CONFIG mode and saving with a normal mode selected

## Audio System

The ESP32-C3 has no hardware DAC. Audio is generated via **LEDC PWM** at 40 kHz (inaudible carrier), with the duty cycle varied to produce audible tones. The headphone's own impedance acts as a low-pass filter.

**Signal path:** `GPIO 18 вҶ’ 100 О© series R вҶ’ 10 ВөF AC-coupling cap вҶ’ 3.5mm jack TIP`

**Tone behaviour in SEARCH mode:**

| Condition                              | Audio output                            | Tone freq  |
|----------------------------------------|-----------------------------------------|------------|
| No signal (RSSI < threshold)           | Silence                                 | вҖ”          |
| Weak signal (RSSI вҖ“90 to вҖ“80 dBm)      | Slow beep (1 beep/s)                    | 440 Hz     |
| Medium signal (RSSI вҖ“80 to вҖ“60 dBm)    | Fast beep (4 beeps/s)                   | 880 Hz     |
| Strong signal (RSSI вүҘ вҖ“60 dBm)         | Continuous tone                         | 1760 Hz    |
| Pitch rising                           | Signal getting stronger (metal-detector feel) | Interpolated |

**Audio in BEACON mode:** Morse click stream at 600 Hz вҖ” each dot/dash produces a click through the earphone, useful for debug monitoring of transmission timing.

**Volume control:** 0вҖ“255 PWM duty (default 180 вүҲ 70%). Configurable via dashboard slider; saved to NVS.

**Compatible headphones:** Standard 3.5mm, impedance 16вҖ“600 О©.

## Morse Engine

The Morse engine uses **PARIS standard timing** вҖ” the word "PARIS" takes exactly 50 units, calibrating against the WPM setting.

```
unit_duration_ms = 1200 / WPM

DOT  = 1 unit
DASH = 3 units
intra-character gap (between elements) = 1 unit
inter-character gap (between letters)  = 3 units
word gap (between words)               = 7 units
```

**Supported characters:** AвҖ“Z, 0вҖ“9, space (word gap)

**Example вҖ” "SOS" at 13 WPM (unit = 92 ms):**

```
S  =  В· В· В·     = DOT GAP DOT GAP DOT
O  =  вҲ’ вҲ’ вҲ’     = DASH GAP DASH GAP DASH
S  =  В· В· В·     = DOT GAP DOT GAP DOT

Full: В·В·В·   вҲ’вҲ’вҲ’   В·В·В·
Time: ~2730 ms
```

**TX can be interrupted mid-character** by pressing SW_MODE. The `transmitMessage()` function checks the interrupt flag between characters.

## Antenna Guide

| Frequency  | Вј-wave length | ВҪ-wave length |
|------------|---------------|---------------|
| 433.5 MHz  | **17.3 cm**   | 34.6 cm       |
| 868 MHz    | 8.6 cm        | 17.3 cm       |
| 915 MHz    | 8.2 cm        | 16.4 cm       |

**Best practices:**

- Use stiff copper wire (AWG 22вҖ“26), vertical orientation
- Keep antenna away from the battery and metal enclosure
- A ground plane (foil on the back of the PCB) improves gain by ~3 dBi
- Helical coil antenna: wind 17.3 cm of wire into a helix (~5 mm diameter) to reduce physical length to ~6 cm with ~1вҖ“2 dBi gain loss

## Troubleshooting

| Symptom                         | Likely cause                      | Fix                                                      |
|---------------------------------|-----------------------------------|----------------------------------------------------------|
| `[ERROR] beginFSK failed: -2`   | SPI wiring error                  | Check CS/SCK/MOSI/MISO pin assignments                   |
| `[ERROR] beginFSK failed: -7`   | Bad solder joint on RA-02         | Reflow SPI pads on RA-02 module                          |
| `[WARN] NVS empty`              | First boot / after factory reset  | Normal вҖ” configure via dashboard                         |

## FAQ

**Q: Can a regular hiker use this without radio knowledge?**  
A: Yes. In BEACON mode the device transmits automatically. In SEARCH mode it scans and alerts you with LED blinks and audio tones вҖ” no radio knowledge needed.

**Q: What receiver do I need to hear the beacon?**  
A: Any AM-mode receiver on 433 MHz: a handheld scanner (Uniden, Baofeng with AM mode), a ham radio transceiver, or an RTL-SDR dongle + SDR# software on a laptop.

**Q: What headphones work with the audio jack?**  
A: Standard 3.5mm wired headphones or earphones, 16вҖ“600 О© impedance. In-ear monitors work best for outdoor use. Wireless/Bluetooth headphones require an adapter or won't work.

**Q: Is this legal to operate?**  
A: 433 MHz ISM band is licence-free in most countries (EU/UK/AU). 915 MHz ISM is licence-free in North America. Always check local regulations before operating. In genuine emergencies, normal frequency restrictions are generally suspended.

**Q: Does it work through snow?**  
A: RF signals at 433 MHz penetrate wet snow with 1вҖ“3 dB/m attenuation. At 1 m burial depth expect ~3вҖ“9 dB signal loss. The +17 dBm output compensates for this. Reduce sleep interval to increase detection probability.

**Q: Can I add GPS coordinates to the message?**  
A: The firmware has a reserved `emergencyGPS` flag. A future update will support appending NMEA coordinates to the Morse message via a connected GPS module.

**Q: How do I update the firmware without reflashing?**  
A: Enter CONFIG mode (hold SW_CONFIG 3 s) and use the OTA update button *(planned for v4.1)*.

**Q: I upgraded from v3.0 вҖ” do I need to factory reset?**  
A: Yes, recommended. The NVS schema changed (new `audioVolume`, `audioEnabled`, `oledEnabled`, `oledInvert` keys). Hold both buttons at boot for 5 s to factory reset, then reconfigure via the dashboard.
